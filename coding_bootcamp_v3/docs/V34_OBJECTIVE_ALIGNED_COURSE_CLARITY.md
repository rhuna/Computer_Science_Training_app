# v35 - Split-Topic Deep Lessons Clarity

v35 rebuilds the course around one rule: every lesson, coaching problem, testing question, coding exercise, verification step, and saved reflection must point to the same learning objective.

## What changed

- Each lesson begins with a clearly asserted objective.
- Lesson content explains the actual topic instead of using generic filler.
- Coaching contains a single coding problem plus a solution walkthrough.
- Testing contains the lesson-matched knowledge check and the coding attempt.
- Verification proves the same objective through real behavior checks.
- Save/reflection records what objective was learned and what evidence proved it.

## Learner flow

1. **Lesson** - Learn one objective with explanation and example.
2. **Coaching** - Study one connected coding problem and solution path.
3. **Testing** - Answer a targeted question and write code.
4. **Verification** - Run behavior checks against the objective.
5. **Save** - Store the finished lesson and reflection.

## Side panel separation

- Left panel: available lessons and course path.
- Center panel: only the current required task.
- Right panel: finished lessons and saved work.

## Curriculum alignment

The curriculum now uses explicit objective-linked modules for problem decomposition, variables, functions, strings/maps, Big-O, search/sort, stacks/queues, trees/graphs, recursion/DP, systems, databases, APIs, testing, security, cloud, UI/UX, language fluency, distributed systems, compilers, and AI/RAG systems.
