# CS Bootcamp Desktop v35 - Coaching Problem Walkthrough

v35 corrects the learning flow so coaching is no longer mixed with quiz questions.

## Flow

1. Lesson - read the topic explanation and examples.
2. Coaching - receive one clearly defined coding problem and a solution walkthrough.
3. Testing - answer lesson-matched questions and write the coding attempt.
4. Verification - run real tests and interpret failures.
5. Save - record the finished lesson and move forward.

## Coaching rules

The coaching stage should not quiz the learner. It should help the learner solve the problem by explaining:

- the problem statement
- inputs and outputs
- constraints
- edge cases
- useful data structures
- algorithm steps
- common mistakes
- complexity expectations

Questions belong in Testing and Verification so the learner first receives help, then gets assessed.
