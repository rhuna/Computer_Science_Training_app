# v35 - Split-Topic Deep Lessons

This version fixes bundled lessons by splitting them into smaller individual learning units. A lesson should not ask the learner to learn hash maps, sets, and unordered sets at the same time. Each concept now receives its own objective, explanation, coaching problem, test question, exercise prompt, and verification target.

## Lesson design rule

Each lesson answers one question: what is the exact concept the learner must understand and prove?

## Examples of splits

- Hash Maps: key-value counting and lookup
- Sets: uniqueness and membership
- Unordered Sets: average O(1) duplicate detection
- SQL Tables: schema, keys, relationships
- SQL Queries: SELECT, WHERE, JOIN, GROUP BY
- Networking: failure categories and client/server model
- HTTP APIs: request/response design and status codes

## Coaching design

The coaching panel contains a coding problem and a solution walkthrough only. It is not a quiz. The questions belong in testing and verification.
