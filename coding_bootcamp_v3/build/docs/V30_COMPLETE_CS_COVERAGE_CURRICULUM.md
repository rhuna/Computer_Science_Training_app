# CS Bootcamp Desktop v31 - Complete CS Coverage Curriculum

v31 expands the course catalog so the simplified one-task-at-a-time interface still produces a well-rounded computer science student. The UI remains intentionally focused: one lesson, one coaching step, one test, one verification step, and one saved review at a time. The breadth is in the lesson catalog, not in extra clutter.

## Coverage areas added

- Mathematical foundations: logic, proofs, invariants, sets, counting, probability, statistics, and linear algebra basics.
- Computer architecture and operating systems: CPU, cache, RAM, processes, threads, scheduling, synchronization, filesystems, virtual memory, and system calls.
- Networking and internet systems: TCP/IP, DNS, ports, HTTP, TLS, APIs, retries, timeouts, and reliability.
- Data modeling and storage: relational design, normalization, transactions, indexes, caches, search, document stores, and analytics stores.
- Software engineering quality: requirements, design docs, acceptance criteria, testing pyramid, debugging, refactoring, and documentation.
- HCI, product thinking, and accessibility: workflows, progressive disclosure, feedback, forms, errors, keyboard access, contrast, and inclusive design.
- Security, privacy, ethics, and responsible engineering: privacy-by-design, abuse cases, defensive controls, audit logs, and responsible AI.
- Cloud, DevOps, deployment, and observability: Linux/shell basics, containers, CI/CD, packaging, logs, metrics, traces, alerts, and incidents.

## Why this matters

A well-rounded student should not only know syntax. They should understand:

1. How computers execute programs.
2. How data is represented, stored, queried, moved, and protected.
3. How algorithms are reasoned about and verified.
4. How real systems fail and recover.
5. How to build software people can use.
6. How to test, maintain, document, deploy, and monitor professional applications.
7. How to think ethically about privacy, security, and automation.

## Interface rule

v31 keeps the app simple. These topics appear as available lessons in the left lesson panel, but the center workspace still reveals only the current required step:

`Lesson -> Coaching -> Testing -> Verification -> Save -> Next Lesson`

Completed items continue to appear separately in the finished-lesson panel so the learner can review or edit past work without cluttering the current task.
