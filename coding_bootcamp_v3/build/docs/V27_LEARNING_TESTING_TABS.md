# V27 - Learning / Testing Tabs

V27 keeps the simplified bootcamp experience from v27, but divides the workflow into two clean tabs instead of one long crowded page.

## Tab 1: Learning + Coaching

This tab is for understanding the topic before writing code.

It contains:

- Module selector
- Language selector
- Lesson list
- Detailed lesson explanation
- Advanced Helper timer
- Coaching pop-ups
- Problem-solving pattern guide
- Algorithm mental model stepper

The learner should use this tab to answer the same kinds of questions a strong interviewer would ask:

1. What is the problem asking?
2. What are the inputs and outputs?
3. What assumptions and edge cases matter?
4. What data structure or pattern fits?
5. What small example proves the plan?

## Tab 2: Testing + Verification

This tab is for proving the solution works.

It contains:

- Connected exercise list
- Code editor
- Build / compile button
- Run button
- Language test button
- Real correctness verification button
- Console output
- Quiz checkpoint
- Progress evidence panel

The learner should use this tab to move from idea to proof:

1. Implement the smallest correct version.
2. Run the code.
3. Verify against real test cases.
4. Fix failed cases.
5. Pass the quiz checkpoint.
6. Record proof of progress.

## Why this version exists

V26 put everything into one guided screen. That was easier than many disconnected tabs, but it could still feel dense.

V27 keeps the course simple while separating the two mental modes:

- Learning mode: understand and plan.
- Testing mode: code and prove correctness.

This is closer to how professional engineers work: think clearly first, then verify with evidence.
