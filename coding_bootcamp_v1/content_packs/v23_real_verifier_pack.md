# v23 Real Verifier Pack

This pack upgrades the course philosophy from marker-based completion to evidence-based correctness.

A learner should not mark an exercise complete because it printed a magic string. A learner should mark an exercise complete when the program can survive multiple inputs and produce the expected output.

## Learner workflow

- Read the lesson objective.
- Study the checklist.
- Open the connected exercise.
- Write or revise the source code.
- Run the real verifier.
- Read failures carefully.
- Fix the code.
- Re-run until all cases pass.
- Review the attempt in Progress Database.
