# v8 Portfolio Builder

v8 adds a portfolio center so the bootcamp is not only a learning tool. It now helps convert lessons, exercises, and generated projects into proof of skill.

## Goals

- Help the learner decide whether a project is GitHub-ready.
- Connect project work to resume language.
- Encourage clean setup, tests, screenshots, README quality, and roadmap discipline.
- Export a markdown checklist that can live beside the project or be copied into a portfolio repo.

## Quality Checklist

The checklist scores projects across eight practical professional criteria:

1. Runs from a clean checkout.
2. README explains problem, setup, usage, and screenshots.
3. Code is organized into small units.
4. Has at least one test or validation command.
5. Handles errors intentionally.
6. Explains architecture or data structure decisions.
7. Uses small professional commits.
8. Includes a clear next-feature roadmap.

## Exports

The Portfolio Builder writes `portfolio_exports/portfolio_checklist_v8.md` beside the executable.

## Future Upgrade

v9 should add Git, testing, CI/CD, and deployment labs so portfolio projects can move from local demos to professional repos with validation pipelines.
